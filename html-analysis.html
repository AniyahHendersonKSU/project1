Topic 01

HTML Analysis: Structure & Semantics

The Cafe template is a short HTML file, no separate pages, but every structural decision is visible in a single scroll. This page focuses on three things: how it uses (and doesn't use) HTML5 semantic elements, how id attributes are doing double duty as both navigation anchors and JavaScript hooks, and how the document is organized top to bottom.

1. Semantic elements: present, but sparse

HTML5 introduced elements like <header>, <footer>, <nav>, <section>, and <article> specifically so a browser, screen reader, or search engine can understand a page's structure instead of guessing from <div> soup. The template uses exactly two of these:

That's the extent of it. The top navigation bar, which is functionally and visually a nav bar, is built as a bare <div class="w3-top"> instead of <nav>. Each content block — About, Menu, Where to Find Us — is its own <div class="w3-container"> rather than a <section>. This isn't a functional bug because visually nothing changes.

Takeaway: HTML5 semantic tags are additive, not automatic. Using W3.CSS (or any CSS framework) doesn't give you semantics for free because you still have to choose <nav> over <div> yourself.

2. IDs doing double duty

Every major section carries an id — home, about, menu, where — and the top nav links to them with plain anchor links (href="#menu"). That's a textbook use of IDs for in-page navigation, and it works without any JavaScript at all.

But the template also repurposes the same mechanism for the Eat/Drink tab switcher: id="Eat" and id="Drinks" are targeted directly by document.getElementById() in the inline <script>. So id is simultaneously an anchor target for the browser and a hook for JavaScript. That's common practice, but it also means the two concerns which are navigation and behavior are entangled in the same attribute, which can get confusing to maintain as a page grows past this size.

3. Document organization: linear and shallow

The whole page is one long linear flow: top bar, header, about, menu, contact form, footer, script. There's no nesting deeper than about four levels almost anywhere, which keeps the markup easy to scan — a real advantage of a template built to be copy-pasted and edited by beginners. The tradeoff is that all styling responsibility falls on class names rather than structure; the HTML alone doesn't tell you that .menu means "hidden unless active tab," you'd have to check the CSS.

4. Two critiques worth flagging

  • Heading hierarchy is broken. Every section title ("ABOUT THE CAFE", "THE MENU") is an <h5>, and there is no <h1> anywhere in the document, the big "the Cafe" text in the header is just a styled <span>. That's chosen purely for the font-size W3.CSS gives h5 by default, not because these are genuinely five levels deep. It looks right, but it's the wrong tool for the job semantically.
  • No alt text on any image. Three <img> elements (coffee shop photo, coffeehouse photo, map) all ship without alt attributes, so a screen reader has nothing to announce for any of them, a real accessibility gap in an otherwise clean template.
