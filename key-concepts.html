Key Concepts

Seven concepts that come up repeatedly across the HTML and CSS analysis pages, explained on their own here for quick reference.

Semantic HTML

Using elements for what they mean, not just how they render. <nav> and <section> tell a browser or screen reader what a block of content is, while <div> says nothing at all.

CSS Selector Specificity

The rule set that decides which conflicting CSS rule wins: inline styles beat IDs, IDs beat classes, classes beat plain element selectors. Higher-specificity rules are harder to override later.

The Box Model

Every element is a box made of content, padding, border, and margin, in that order from inside out. box-sizing: border-box changes how width is calculated so padding and borders don't enlarge the box.

Utility-First CSS

A framework style (used by W3.CSS, Tailwind, and others) where small, single-purpose classes like .w3-padding are composed directly in HTML, instead of writing one custom class per component.

Flexbox

A one-dimensional CSS layout mode for arranging items in a row or column, distributing space between them, and aligning them, which is used on this site's navigation bar.

CSS Grid

A two-dimensional layout system for arranging items into rows and columns at once, which is used on this site for the overall page structure and the key-concepts card layout above.

Responsive Design

Building a page so it adapts to different screen sizes, usually via a viewport meta tag plus media queries or flexible units (%, fr, auto-fit) instead of fixed pixel widths.
